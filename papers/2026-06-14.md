# Daily Papers — 2026-06-14

> Topics: LLM for Code, LLM for Software Engineering

## Daily Highlight

Today's papers reveal a strong focus on **LLM-based agents** for complex, dynamic environments and scientific discovery, with particular emphasis on improving agent reasoning, tool orchestration, and multi-agent system design for software engineering tasks. Papers on agent environments, knowledge orchestration, and test-time scaling suggest the field is moving beyond single-step reasoning toward more sophisticated agent architectures capable of handling real-world complexity. Additionally, there is growing attention to **robustness and reproducibility**—detecting reasoning failures and automating quality assessment—indicating maturation of LLM evaluation practices.

---

## Papers by Theme

### Agents & Agent Architectures
- **[EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments](https://arxiv.org/abs/2606.13681v1)** — Introduces a benchmark for evaluating LLM agents in dynamic environments where tasks and conditions continually change, addressing the gap between static benchmarks and real-world deployment.
- **[InterleaveThinker: Reinforcing Agentic Interleaved Generation](https://arxiv.org/abs/2606.13679v1)** — Enables agents to perform interleaved text-image generation for applications like visual narratives and embodied manipulation, overcoming architectural limitations of single-image generators.
- **[HyperTool: Beyond Step-Wise Tool Calls for Tool-Augmented Agents](https://arxiv.org/abs/2606.13663v1)** — Proposes an execution model that bundles deterministic tool workflows into higher-level abstractions, reducing context overhead and improving agent reasoning by addressing execution-granularity mismatch.
- **[SpatialClaw: Rethinking Action Interface for Agentic Spatial Reasoning](https://arxiv.org/abs/2606.13673v1)** — Studies how the action interface constrains vision-language agent effectiveness for spatial reasoning tasks, proposing improvements to how tool-augmented agents invoke specialist perception modules.

### Agent Knowledge & Reasoning
- **[Agents-K1: Towards Agent-native Knowledge Orchestration](https://arxiv.org/abs/2606.13669v1)** — Develops an end-to-end system for orchestrating scientific knowledge in LLM-based research agents by capturing key entities, claims, evidence, and method lineages from scientific literature.
- **[Learning to Reason by Analogy via Retrieval-Augmented Reinforcement Fine-Tuning](https://arxiv.org/abs/2606.13680v1)** — Proposes analogy-based retrieval for complex reasoning tasks, moving beyond lexical/semantic similarity to identify structurally similar problems that share reasoning strategies.
- **[Operadic consistency: a label-free signal for compositional reasoning failures in LLMs](https://arxiv.org/abs/2606.13649v1)** — Introduces operadic theory-based diagnostics to detect compositional reasoning failures in LLMs at inference time without ground-truth labels.

### Scientific Discovery & Autonomous Systems
- **[EurekAgent: Agent Environment Engineering is All You Need For Autonomous Scientific Discovery](https://arxiv.org/abs/2606.13662v1)** — Argues that the bottleneck for LLM-based scientific discovery is shifting from model capability to environment design, proposing agent environment engineering as the key to scaling autonomous discovery.
- **[WEAVER, Better, Faster, Longer: An Effective World Model for Robotic Manipulation](https://arxiv.org/abs/2606.13672v1)** — Develops world models for robotics that jointly optimize fidelity, efficiency, and horizon length to enable policy evaluation, improvement, and test-time planning with limited real-world interaction.
- **[RepWAM: World Action Modeling with Representation Visual-Action Tokenizers](https://arxiv.org/abs/2606.13674v1)** — Presents representation-centric world action models using visual-action tokenizers that improve instruction-following dynamics beyond pixel-reconstruction approaches.

### Software Engineering with LLMs (Multi-Agent & Agents)
- **[Trae Agent: An LLM-based Agent for Software Engineering with Test-time Scaling](https://www.semanticscholar.org/paper/15a9b53dc8313eb0542dac00e1ea6e3dad2bbf51)** — Introduces test-time scaling techniques for LLM agents to improve software issue resolution through ensemble reasoning approaches.
- **[LLM-Based Multi-Agent Systems for Software Engineering: Literature Review, Vision, and the Road Ahead](https://www.semanticscholar.org/paper/027d9a3371c13ea567bca75e675db295022a300b)** — Surveys the integration of LLMs into multi-agent systems for software engineering, exploring their potential to address complex SE challenges through collaborative reasoning.
- **[From LLMs to LLM-based Agents for Software Engineering: A Survey of Current, Challenges and Future](https://www.semanticscholar.org/paper/987c5ab507b2fce644a9b73a8f8338c80cba944d)** — Comprehensive survey of LLM applications in software engineering, from code generation to vulnerability detection, highlighting limitations and the emergence of agent-based approaches.
- **[Nemotron-CORTEXA: Enhancing LLM Agents for Software Engineering Tasks via Improved Localization and Solution Diversity](https://www.semanticscholar.org/paper/34d96fe3d9c45a47d9021cc20557a054a35df935f)** — Enhances LLM agents for SE tasks by improving bug localization and generating diverse solution candidates.
- **[Designing LLM-based Multi-Agent Systems for Software Engineering Tasks: Quality Attributes, Design Patterns and Rationale](https://www.semanticscholar.org/paper/4fe99230d170b6c17570ec1cb4b10c3e99aef56e)** — Proposes design patterns and quality attributes for architecting LLM-based multi-agent systems tailored to software engineering tasks.

### Code & Software Security
- **[Large Language Model (LLM) for Software Security: Code Analysis, Malware Analysis, Reverse Engineering](https://www.semanticscholar.org/paper/605a8b0d425a7cf6a0e97eafa681e656081d958e)** — Reviews LLM applications in cybersecurity including malware detection, code analysis, and automated threat analysis with focus on novel malware variant identification.

### LLM Evaluation & Quality Assessment
- **[Automated Validation of LLM-based Evaluators for Software Engineering Artifacts](https://www.semanticscholar.org/paper/01b881d5a3471d4c0270b07c77604b9ebdf5fc95)** — Introduces REFINE, an automated method to validate LLM-based evaluators for code and SE artifacts, addressing the challenge of scalable and fine-grained quality assessment without costly human evaluation.
- **[Automated reproducibility assessments in the social and behavioral sciences using large language models](https://arxiv.org/abs/2606.13670v1)** — Demonstrates that LLMs can automate reproducibility assessments in research, scaling what is typically done by independent researchers manually.

### Data Attribution & Model Analysis
- **[Influcoder: Distilling Decoders' Gradient Influence Rankings into an Encoder for Data Attribution](https://arxiv.org/abs/2606.13668v1)** — Proposes a method to distill gradient influence rankings into an encoder for efficient data attribution, enabling identification of training samples that influence model outputs without expensive per-sample analysis.
