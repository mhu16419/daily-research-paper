# Daily Papers — 2026-06-19

> Topics: LLM for Code, LLM for Software Engineering

## Daily Highlight

Today's collection reveals a strong focus on **LLM-based agents and repository-scale software engineering**, with papers addressing practical challenges like coding agent guidance, vulnerability detection, and multi-language code benchmarking. Key trends include moving beyond contamination concerns toward interpretable evaluation frameworks, expanding benchmark coverage to multiple programming languages, and improving agent performance through repository-aware context injection.

## Papers by Theme

### Benchmarks & Evaluation
- **[Multi-LCB: Extending LiveCodeBench to Multiple Programming Languages](https://arxiv.org/abs/2606.20517v1)** — Extends the widely-adopted LiveCodeBench to support multiple programming languages beyond Python, providing contamination-aware evaluation for LLM coding capability.
- **[Calibration Without Comprehension: Diagnosing the Limits of Fine-Tuning LLMs for Vulnerability Detection in Systems Software](https://arxiv.org/abs/2606.20502v1)** — Introduces CWE-Trace, a framework with 834 manually curated samples and strict temporal splits to assess whether LLMs genuinely reason about security vulnerabilities or merely pattern-match.

### Agents & Repository-Scale Tasks
- **[Code Graph Model (CGM): A Graph-Integrated Large Language Model for Repository-Level Software Engineering Tasks](https://www.semanticscholar.org/paper/13b3bb88f7f28b76214e13e4792c159b8b75cedf)** — Proposes a graph-integrated LLM that combines code structure awareness with language models to improve performance on repository-level engineering tasks using open-source models.
- **[Probe-and-Refine Tuning of Repository Guidance for Coding Agents](https://arxiv.org/abs/2606.20512v1)** — Develops methods for tuning repository guidance (AGENTS.md instructions) that help LLM-based coding agents navigate codebases more effectively.
- **[Large Language Model-Based Agents for Software Engineering: A Survey](https://www.semanticscholar.org/paper/335589d44ca3067758822d84f83f7d3e194cb61e)** — Comprehensive survey of LLM-based agents applied to software engineering, examining how they extend LLM capabilities through external tools and resources.

### Vulnerability & Security Analysis
- **[Large Language Model (LLM) for Software Security: Code Analysis, Malware Analysis, Reverse Engineering](https://www.semanticscholar.org/paper/605a8b0d425a7cf6a0e97eafa681e656081d958e)** — Reviews LLM applications in cybersecurity, covering malware detection, code analysis, and real-time threat monitoring capabilities.

### Program Repair & Code Generation
- **[A Critical Review of Large Language Model on Software Engineering: An Example from ChatGPT and Automated Program Repair](https://www.semanticscholar.org/paper/b33ee4c8c707db84fc0cf8166a9d8e3ae69e3378)** — Critical examination of LLM performance on software engineering tasks including automated program repair, code summarization, and completion, using ChatGPT as a case study.
- **[Large Language Model Assisted Software Engineering: Prospects, Challenges, and a Case Study](https://www.semanticscholar.org/paper/e7b7ba8689cd200082ac07c66e1898d71aff46b1)** — Explores prospects and challenges of applying LLMs to software engineering with practical case studies.

### Safety & Alignment
- **[What Do Safety-Aligned LLMs Learn From Mixed Compliance Demonstrations?](https://arxiv.org/abs/2606.20508v1)** — Investigates how safety-aligned LLMs interpret and learn from mixed compliance demonstrations (benign vs. harmful), testing whether models can be jailbroken through demonstration context.
- **[From Efficiency to Leakage -- Privacy Backdoor in Federated Language Model Fine-Tuning](https://arxiv.org/abs/2606.20553v1)** — Reveals privacy vulnerabilities in federated learning of language models, showing how parameter-efficient fine-tuning (PEFT) adapters can leak sensitive training data.

### Interpretability & Reasoning
- **[How Transparent is DiffusionGemma?](https://arxiv.org/abs/2606.20560v1)** — Analyzes reasoning transparency in DiffusionGemma by decomposing transparency into variable and computation transparency, examining how continuous latent-space computation affects interpretability.

### Accessibility & Tools
- **[Easy Reads: A Python program for making Scientific Papers on arXiv more Reader Friendly and Accessible](https://arxiv.org/abs/2606.20550v1)** — Introduces a Python tool to improve readability and accessibility of dense scientific papers through formatting enhancements.
