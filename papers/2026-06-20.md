# Daily Papers — 2026-06-20

> Topics: LLM for Code, LLM for Software Engineering

# Daily Highlight

Repository-level code understanding and agent-based software engineering emerge as dominant themes, with papers introducing graph-integrated LLMs for whole-codebase tasks and frameworks for improving coding agent reliability through better context management. Security and vulnerability detection in LLMs are receiving increased scrutiny, with studies exposing limitations in fine-tuned models' reasoning capabilities and backdoor risks in federated learning scenarios. Multi-language benchmarking and interpretability of LLM reasoning are also gaining traction, reflecting broader efforts to standardize evaluation and understand model decision-making in SE tasks.

## Papers by Theme

### Benchmarks & Evaluation
- **[Multi-LCB: Extending LiveCodeBench to Multiple Programming Languages](https://arxiv.org/abs/2606.20517v1)** — Expands the LiveCodeBench coding benchmark beyond Python to multiple programming languages for more comprehensive LLM evaluation.
- **[Calibration Without Comprehension: Diagnosing the Limits of Fine-Tuning LLMs for Vulnerability Detection in Systems Software](https://arxiv.org/abs/2606.20502v1)** — Introduces CWE-Trace, a contamination-aware framework with strict temporal splits to evaluate whether LLMs genuinely reason about security vulnerabilities or merely pattern-match.

### Agents & Agentic Systems
- **[Probe-and-Refine Tuning of Repository Guidance for Coding Agents](https://arxiv.org/abs/2606.20512v1)** — Develops methods to optimize AGENTS.md instructions for improving LLM-based coding agents' understanding of repository structure and workflows.
- **[Large Language Model-Based Agents for Software Engineering: A Survey](https://www.semanticscholar.org/paper/335589d44ca3067758822d84f83f7d3e194cb61e)** — Comprehensive survey of LLM-based agents in software engineering, examining their capabilities for utilizing external resources and tools.
- **[Execution-State Capsules: Graph-Bound Execution-State Checkpoint and Restore for Low-Latency, Small-Batch, On-Device Physical-AI Serving](https://arxiv.org/abs/2606.20537v1)** — Proposes checkpoint/restore mechanisms for execution state beyond KV caches, enabling efficient on-device LLM agents for interactive applications.

### Code Generation & Repository-Level Tasks
- **[Code Graph Model (CGM): A Graph-Integrated Large Language Model for Repository-Level Software Engineering Tasks](https://www.semanticscholar.org/paper/13b3bb88f7f28b76214e13e4792c159b8b75cedf)** — Introduces a graph-integrated LLM architecture that combines code structure with language models for whole-repository code generation and software engineering tasks.

### Program Repair & Bug Fixing
- **[A Critical Review of Large Language Model on Software Engineering: An Example from ChatGPT and Automated Program Repair](https://www.semanticscholar.org/paper/b33ee4c8c707db84fc0cf8176a9d8e3ae69e3378)** — Critical analysis of LLM performance on automated program repair, examining ChatGPT's effectiveness and limitations in fixing code bugs.

### Security & Safety
- **[What Do Safety-Aligned LLMs Learn From Mixed Compliance Demonstrations?](https://arxiv.org/abs/2606.20508v1)** — Investigates how safety-aligned LLMs interpret mixed compliance demonstrations (benign and harmful requests), revealing potential vulnerabilities in alignment mechanisms.
- **[From Efficiency to Leakage -- Privacy Backdoor in Federated Language Model Fine-Tuning](https://arxiv.org/abs/2606.20553v1)** — Demonstrates privacy backdoor attacks in parameter-efficient fine-tuning (PEFT) of LLMs in federated learning settings, exposing data leakage risks.
- **[Large Language Model (LLM) for Software Security: Code Analysis, Malware Analysis, Reverse Engineering](https://www.semanticscholar.org/paper/605a8b0d425a7cf6a0e97eafa681e656081d958e)** — Survey of LLM applications in cybersecurity including malware detection, code analysis, and reverse engineering capabilities.

### Model Interpretability & Reasoning
- **[How Transparent is DiffusionGemma?](https://arxiv.org/abs/2606.20560v1)** — Analyzes reasoning transparency in models that compute in latent spaces, decomposing transparency into variable and computational components to understand decision-making.

### General SE & Code Understanding
- **[Large Language Model Assisted Software Engineering: Prospects, Challenges, and a Case Study](https://www.semanticscholar.org/paper/e7b7ba8689cd200083ac07c66e1898d71aff46b1)** — Comprehensive case study examining prospects and challenges of integrating LLMs into software engineering workflows.
- **[Easy Reads: A Python program for making Scientific Papers on arXiv more Reader Friendly and Accessible](https://arxiv.org/abs/2606.20550v1)** — Develops a tool for reformatting arXiv papers to improve readability and accessibility, relevant to knowledge dissemination in SE research.
