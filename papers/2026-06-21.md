# Daily Papers — 2026-06-21

> Topics: LLM for Code, LLM for Software Engineering

## Daily Highlight

Today's papers reveal a strong focus on **LLM-based agents and repository-level software engineering**, with emerging emphasis on **security vulnerability detection** and **multi-language code benchmarking**. Key trends include moving beyond function-level tasks to whole-repository understanding, integrating graph-based code representations with LLMs, and developing contamination-aware evaluation frameworks to ensure robust capability assessment.

---

## Papers by Theme

### Benchmarks & Evaluation
- **[Multi-LCB: Extending LiveCodeBench to Multiple Programming Languages](https://arxiv.org/abs/2606.20517v1)** — Extends the contamination-aware LiveCodeBench to support multiple programming languages beyond Python for more comprehensive LLM code-generation evaluation.
- **[Calibration Without Comprehension: Diagnosing the Limits of Fine-Tuning LLMs for Vulnerability Detection in Systems Software](https://arxiv.org/abs/2606.20502v1)** — Introduces CWE-Trace, a temporally-split benchmark of 834 Linux kernel samples to assess whether LLMs genuinely reason about vulnerabilities or pattern-match on contaminated data.

### Agents & Repository-Level Tasks
- **[Code Graph Model (CGM): A Graph-Integrated Large Language Model for Repository-Level Software Engineering Tasks](https://www.semanticscholar.org/paper/13b3bb88f7f28b76214e13e4792c159b8b75cedf)** — Proposes a graph-integrated LLM that combines code structure with language models to tackle repository-level SE tasks beyond function-level code generation.
- **[Large Language Model-Based Agents for Software Engineering: A Survey](https://www.semanticscholar.org/paper/335589d44ca3067758822d84f83f7d3e194cb61e)** — Comprehensive survey of LLM-based agents in SE, covering how agents extend LLMs with external tools and resources.
- **[Probe-and-Refine Tuning of Repository Guidance for Coding Agents](https://arxiv.org/abs/2606.20512v1)** — Develops methods to tune repository context (e.g., AGENTS.md files) to improve LLM-based coding agents' ability to navigate complex codebases.

### Security & Vulnerability Analysis
- **[Large Language Model (LLM) for Software Security: Code Analysis, Malware Analysis, Reverse Engineering](https://www.semanticscholar.org/paper/605a8b0d425a7cf6a0e97eafa681e656081d958e)** — Reviews LLM capabilities for malware detection, code analysis, and threat identification in cybersecurity applications.

### Program Repair & Code Quality
- **[A Critical Review of Large Language Model on Software Engineering: An Example from ChatGPT and Automated Program Repair](https://www.semanticscholar.org/paper/b33ee4c8c707db84fc0cf8162a9d8e3ae69e3378)** — Critical analysis of LLM performance on APR and other SE tasks, examining real-world effectiveness and limitations.

### Safety & Alignment
- **[What Do Safety-Aligned LLMs Learn From Mixed Compliance Demonstrations?](https://arxiv.org/abs/2606.20508v1)** — Studies how in-context demonstrations (benign vs. harmful) influence safety-aligned LLMs' compliance behavior and vulnerability to jailbreaks.
- **[From Efficiency to Leakage -- Privacy Backdoor in Federated Language Model Fine-Tuning](https://arxiv.org/abs/2606.20553v1)** — Reveals privacy leakage risks in federated fine-tuning of LLMs using parameter-efficient methods (PEFT adapters).

### Interpretability & Reasoning
- **[How Transparent is DiffusionGemma?](https://arxiv.org/abs/2606.20560v1)** — Examines transparency and interpretability of LLMs that perform reasoning in continuous latent spaces, addressing debuggability and alignment concerns.

### General SE Applications
- **[Large Language Model Assisted Software Engineering: Prospects, Challenges, and a Case Study](https://www.semanticscholar.org/paper/e7b7ba8689cd200083ac07c66e1898d71aff46b1)** — Overview of LLM applications in SE, discussing opportunities and practical challenges.

### LLM Infrastructure & Serving
- **[Execution-State Capsules: Graph-Bound Execution-State Checkpoint and Restore for Low-Latency, Small-Batch, On-Device Physical-AI Serving](https://arxiv.org/abs/2606.20537v1)** — Proposes efficient execution-state management for on-device LLM agents and interactive systems beyond traditional KV-cache methods.
