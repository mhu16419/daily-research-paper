# Daily Highlight Research Summary

## Daily Highlight

Today's collection reveals a strong focus on **LLM benchmarking and evaluation** with papers extending existing frameworks to multiple languages and stricter contamination controls, alongside **practical systems challenges** for deploying coding agents in real repositories. Key trends include the emergence of repository-aware guidance mechanisms for LLM-based coding agents and growing attention to security vulnerabilities in systems code, reflecting the field's shift from synthetic benchmarks toward production-readiness and interpretability.

## Papers by Theme

### Benchmarks & Evaluation
- **[Multi-LCB: Extending LiveCodeBench to Multiple Programming Languages](https://arxiv.org/abs/2606.20517v1)** — Extends the widely-adopted LiveCodeBench coding evaluation framework from Python to multiple programming languages while maintaining contamination-aware evaluation.
- **[Calibration Without Comprehension: Diagnosing the Limits of Fine-Tuning LLMs for Vulnerability Detection in Systems Software](https://arxiv.org/abs/2606.20502v1)** — Introduces CWE-Trace, a framework with manually curated Linux kernel samples and strict temporal splits to assess whether LLMs genuinely reason about security vulnerabilities or merely pattern-match on contaminated data.

### Agents & Repository Guidance
- **[Probe-and-Refine Tuning of Repository Guidance for Coding Agents](https://arxiv.org/abs/2606.20512v1)** — Proposes methods to optimize AGENTS.md instructions for LLM-based coding agents, balancing repository context provision with instruction clarity and reducing unnecessary guidance overhead.

### Safety & Alignment
- **[What Do Safety-Aligned LLMs Learn From Mixed Compliance Demonstrations?](https://arxiv.org/abs/2606.20508v1)** — Investigates how safety-aligned LLMs interpret mixed benign and harmful compliance demonstrations, clarifying mechanisms by which in-context examples affect model behavior during inference.

### Security in LLM Fine-Tuning
- **[From Efficiency to Leakage -- Privacy Backdoor in Federated Language Model Fine-Tuning](https://arxiv.org/abs/2606.20553v1)** — Reveals privacy vulnerabilities in parameter-efficient fine-tuning (PEFT) approaches commonly used in federated learning, demonstrating how adapters can leak training data despite frozen base models.
