# Daily Papers — 2026-06-24

> Topics: LLM for Code, LLM for Software Engineering

## Daily Highlight

Today's collection reveals a strong focus on LLM-based code generation reliability and safety, with emerging work on agentic systems for software tasks and structured diagnostic approaches for repair. Key trends include benchmarking code generation under varied conditions (prompt tone, ethical safety, NFR assessment), improving agent efficiency through better fault localization, and evaluating LLM robustness across real-world scenarios.

## Papers by Theme

### Code Generation & Quality
- **[Can LLM Replace Stack Overflow? A Study on Robustness and Reliability of Large Language Model Code Generation](https://www.semanticscholar.org/paper/b61b260de1599e6e89491cad9160898fcd3b34c2)** — Investigates the reliability and robustness of LLM-generated code compared to traditional resources like Stack Overflow.

- **[A Deep Dive Into Large Language Model Code Generation Mistakes: What and Why?](https://www.semanticscholar.org/paper/2d797aa9a4127eb49a8b565ad9da4d3ac0727e1b)** — Analyzes types and root causes of errors in LLM code generation for real-world software development scenarios beyond standalone functions.

- **[ChartCoder: Advancing Multimodal Large Language Model for Chart-to-Code Generation](https://www.semanticscholar.org/paper/6cf64f544140a2f27bc0851fee2506aaf1e58723)** — Extends multimodal LLMs to generate code from chart images by parsing visual information losslessly into executable representations.

### Code Generation - Prompt Engineering & Optimization
- **[DDPT: Diffusion-Driven Prompt Tuning for Large Language Model Code Generation](https://www.semanticscholar.org/paper/d260e162b658b9ce3ae20ad801f9fcd1a9cfdea4)** — Automates prompt engineering for LLM code generation using diffusion-based tuning to improve code quality without manual effort.

- **[PromptTone: A Dataset for Evaluating Large Language Model Code Generation Under Varying Prompt Politeness Levels](https://www.semanticscholar.org/paper/8218c0cff581ee9a5a81a31b411c0ce4826e7fde)** — Introduces a controlled dataset examining how prompt politeness and communicative tone influence LLM code generation quality.

### Code Safety & Security
- **[CodeSafetyBench evaluating and improving ethical safety in large language model code generation](https://www.semanticscholar.org/paper/42b931ede3cadbbe5b9d44bab66f36f2b0deb471)** — Benchmarks ethical and safety risks in LLM code generation with 1,050 harmful cases and proposes defense strategies.

- **[Embedding Traceability in Large Language Model Code Generation: Towards Trustworthy AI-Augmented Software Engineering](https://www.semanticscholar.org/paper/4013bbb934068b9d01dd977e2382979f751a4865)** — Addresses requirement traceability challenges in LLM-generated code to enable trustworthy AI-augmented development.

### Program Repair & Fault Localization
- **[SHERLOC: Structured Diagnostic Localization for Code Repair Agents](https://arxiv.org/abs/2606.24820v1)** — Improves LLM agent efficiency in repository-level code repair by providing structured diagnostic context alongside fault locations.

### Software Engineering - Requirements & Assessment
- **[Accuracy and Satisfaction in Multi-Turn LLM Dialogues for NFR Assessment](https://arxiv.org/abs/2606.24834v1)** — Evaluates LLM dialogue assistants' accuracy in handling Non-Functional Requirements, filling a gap beyond functional correctness assessment.

### Code Evaluation & Autograding
- **[Evaluating Large Language Model Code Generation as an Autograding Mechanism for "Explain in Plain English" Questions](https://www.semanticscholar.org/paper/f0201796ff71eb37100057776995ac62db242011)** — Explores using LLMs to automatically grade student explanations of code comprehension in programming courses.

### Agentic Systems
- **[Grading the Grader: Lessons from Evaluating an Agentic Data Analysis System](https://arxiv.org/abs/2606.24839v1)** — Examines challenges in evaluating agentic systems that produce complex outputs (code, results, diagnostics) and distinguishing genuine errors from grading artifacts.

- **["Zooming In" on Agentic Web Browsers as Assistive Technologies: A Case Study with a Low-Vision Technology Expert](https://arxiv.org/abs/2606.24870v1)** — Studies LLM-powered agentic web browsers as assistive technologies for visually-impaired users through a case study approach.

### General LLM Capabilities & Benchmarks
- **[HelpBench: Assessing the Ability of LLMs to Provide Privacy, Safety, and Security Advice](https://arxiv.org/abs/2606.24819v1)** — Introduces a benchmark with 450 curated questions to evaluate LLM accuracy in providing digital privacy, safety, and security guidance.

- **[Matching Tasks to Objectives: Fine-Tuning and Prompt-Tuning Strategies for Encoder-Decoder Pre-trained Language Models](https://arxiv.org/abs/2606.24841v1)** — Studies how diverse pre-training objectives affect encoder-decoder model performance on generation and question-answering tasks.
