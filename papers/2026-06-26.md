# Daily Papers — 2026-06-26

> Topics: LLM for Code, LLM for Software Engineering

# Daily Highlight

Today's collection reveals a fragmented landscape where only a handful of papers directly address LLM applications in software engineering, while the majority focus on unrelated domains (physics, quantum computing, cooling systems, etc.). The few relevant papers highlight emerging trends: **graph-integrated LLMs for repository-level tasks, LLM-based agents as a unifying paradigm for SE work, and growing emphasis on empirical rigor and reproducibility standards** when evaluating LLMs in software engineering contexts.

---

## Papers by Theme

### Agents & Multi-Tool Systems
- **[Large Language Model-Based Agents for Software Engineering: A Survey](https://www.semanticscholar.org/paper/335589d44ca3067758822d84f83f7d3e194cb61e)** — Comprehensive survey examining how LLM-based agents extend capabilities through external tools and resources for diverse SE tasks.
- **[CHIA: An open-source framework for principled, agentic AI-driven hardware/software co-design research](https://arxiv.org/abs/2606.27350v1)** — Open-source framework enabling agentic AI systems for hardware/software co-design across architecture, systems, and compilers.

### Repository-Level Code Understanding & Generation
- **[Code Graph Model (CGM): A Graph-Integrated Large Language Model for Repository-Level Software Engineering Tasks](https://www.semanticscholar.org/paper/13b3bb88f7f28b76214e13e4792c159b8b75cedf)** — Graph-integrated LLM approach enabling open-source models to handle complex, repository-level software engineering tasks beyond function-level generation.

### Program Repair & Code Quality
- **[Large Language Model (LLM) for Software Security: Code Analysis, Malware Analysis, Reverse Engineering](https://www.semanticscholar.org/paper/605a8b0d425a7cf6a0e97eafa681e656081d958e)** — Survey of LLM applications in cybersecurity including malware detection, code analysis, and automated threat analysis.
- **[A Critical Review of Large Language Model on Software Engineering: An Example from ChatGPT and Automated Program Repair](https://www.semanticscholar.org/paper/b33ee4c8c707db84fc0cf8176a9d8e3ae69e3378)** — Critical evaluation of ChatGPT and LLMs for automated program repair and other SE tasks, assessing practical effectiveness.

### Information Extraction & Documentation
- **[Data extraction for systematic mapping study using a large language model - a proof-of-concept study in software engineering](https://www.semanticscholar.org/paper/bc8500fc059b5879692bb6a15fad84183cb1afed)** — Demonstrates LLM feasibility for automating labor-intensive data extraction in systematic literature reviews.
- **[LLM-Based Examination of Eligibility Criteria from Securities Prospectuses at the German Central Bank](https://arxiv.org/abs/2606.27316v1)** — Applies LLMs to extract and verify complex eligibility criteria from semi-structured documents, showing practical information extraction use cases.

### Code Generation & Model-Driven Engineering
- **[From Natural Language to Web Applications: Using Large Language Models for Model-Driven Software Engineering](https://www.semanticscholar.org/paper/ba951054c92b8f913d7aafa5dc278060b18e2bae)** — Explores LLM application to model-driven SE for generating web applications from natural language specifications.

### LLM Training & Reward Mechanisms
- **[Reinforcement Learning without Ground-Truth Solutions can Improve LLMs](https://arxiv.org/abs/2606.27369v1)** — Proposes RiVER framework enabling RL training of LLMs on optimization tasks without ground-truth solutions, applicable to open-ended code tasks.

### Decoding & Output Quality
- **[When are likely answers right? On Sequence Probability and Correctness in LLMs](https://arxiv.org/abs/2606.27359v1)** — Investigates fundamental relationship between sequence probability and correctness in LLM outputs, relevant for code generation reliability.

### Empirical Methods & Reproducibility
- **[Guidelines for Empirical Studies in Software Engineering involving Large Language Models](https://www.semanticscholar.org/paper/5e9b3fa83432e3b242ea128037fa5fa0f7eeebf9)** — Establishes taxonomy and best practices for reproducible empirical evaluation of LLMs in SE research, addressing non-determinism and evaluation rigor.
- **[Large Language Model Assisted Software Engineering: Prospects, Challenges, and a Case Study](https://www.semanticscholar.org/paper/e7b7ba8689cd200083ac07c66e1898d71aff46b1)** — Exploratory study examining prospects and practical challenges of integrating LLMs into software engineering workflows.
