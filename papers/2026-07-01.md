# Daily Papers — 2026-07-01

> Topics: LLM for Code, LLM for Software Engineering

## Daily Highlight

LLM agents for software engineering are advancing toward more robust and verifiable solutions, with papers introducing verifier-guided code generation in Dafny, skill composition frameworks, and neuro-symbolic compliance review systems. Key trends include moving beyond simple prompting toward structured, interpretable agent reasoning and the growing importance of long-horizon task decomposition with dense supervision signals. Repository-level code understanding and graph-integrated models are emerging as critical directions to handle complex software engineering workflows beyond function-level tasks.

## Papers by Theme

### Code Generation & Verification
- **[AxDafny: Agentic Verified Code Generation in Dafny](https://arxiv.org/abs/2606.32007v1)** — A verifier-guided repair framework that iteratively generates implementations, invariants, and assertions for provably correct code in Dafny with a new benchmark of 250 competition-style problems.
- **[Code Graph Model (CGM): A Graph-Integrated Large Language Model for Repository-Level Software Engineering Tasks](https://www.semanticscholar.org/paper/13b3bb88f7f28b76214e13e4792c159b8b75cedf)** — Proposes a graph-integrated LLM architecture to enable open-source models to handle repository-level code generation and analysis tasks.

### LLM Agents & Skill Composition
- **[Generative Skill Composition for LLM Agents](https://arxiv.org/abs/2606.32025v1)** — Addresses how LLM agents can compose modular skills (sandboxing, testing, refactoring) across growing libraries and domains for complex task solving.
- **[QVal: Cheaply Evaluating Dense Supervision Signals for Long-Horizon LLM Agents](https://arxiv.org/abs/2606.32034v1)** — Proposes efficient dense supervision methods for scoring intermediate steps in long-horizon agent trajectories to provide richer learning signals than outcome-only rewards.
- **[Large Language Model-Based Agents for Software Engineering: A Survey](https://www.semanticscholar.org/paper/335589d44ca3067758822d84f83f7d3e194cb61e)** — Comprehensive survey of how LLM-based agents extend standalone LLMs with tool use and external resource integration for software engineering tasks.

### Program Repair & Bug Fixing
- **[A Critical Review of Large Language Model on Software Engineering: An Example from ChatGPT and Automated Program Repair](https://www.semanticscholar.org/paper/b33ee4c8c707db84fc0cf8176a9d8e3ae69e3378)** — Reviews ChatGPT and LLM performance on automated program repair, code summarization, and code completion with critical analysis of limitations.

### Security & Compliance
- **[Large Language Model (LLM) for Software Security: Code Analysis, Malware Analysis, Reverse Engineering](https://www.semanticscholar.org/paper/605a8b0d425a7cf6a0e97eafa681e656081d958e)** — Surveys LLM applications in malware detection, code structure analysis, and automated threat analysis for cybersecurity.
- **[PolicyGuard: From Organizational Policies to Neuro-Symbolic Compliance Review Engines](https://arxiv.org/abs/2606.32004v1)** — Presents a neuro-symbolic framework that converts organizational policies into inspectable, testable compliance logic rather than relying on implicit end-to-end LLM prompting.

### LLM Reliability & Reasoning
- **[Reinforcement Learning with Metacognitive Feedback Elicits Faithful Uncertainty Expression in LLMs](https://arxiv.org/abs/2606.32032v1)** — Applies metacognitive reinforcement learning to improve LLMs' ability to express genuine uncertainty and recognize knowledge boundaries.
- **[Introspective Coupling: Self-Explanation Training Tracks Behavioral Change Despite Fixed Supervision](https://arxiv.org/abs/2606.32038v1)** — Shows that training LMs to explain their counterfactual behavior enables faithful introspection even with fixed supervision.
- **[When LLMs Read Tables Carelessly: Measuring and Reducing Data Referencing Errors](https://arxiv.org/abs/2606.32029v1)** — Identifies and analyzes data referencing errors where LLMs misread or omit table values, threatening intermediate reasoning correctness.
- **[Surrogate Fidelity: When Can Open LLMs Explain Closed Ones?](https://arxiv.org/abs/2606.32008v1)** — Investigates when mechanistic interpretability measurements on open models can reliably explain closed proprietary LLMs.

### LLM Evaluation & Methodology
- **[Guidelines for Empirical Studies in Software Engineering involving Large Language Models](https://www.semanticscholar.org/paper/5e9b3fa83432e3b242ea128037fa5fa0f7eeebf9)** — Establishes a taxonomy and guidelines for reproducible empirical studies using LLMs in software engineering research.
- **[Data extraction for systematic mapping study using a large language model - a proof-of-concept study in software engineering](https://www.semanticscholar.org/paper/bc8500fc059b5879692bb6a01f66b15cedf)** — Demonstrates using LLMs (ChatGPT-4o) to accelerate data extraction in systematic mapping studies for software engineering literature.

### Software Engineering Applications
- **[Large Language Model Assisted Software Engineering: Prospects, Challenges, and a Case Study](https://www.semanticscholar.org/paper/e7b7ba8689cd200083ac07c66e1898d71aff46b1)** — Explores prospects and challenges of LLM-assisted software engineering with practical case studies.
- **[From Natural Language to Web Applications: Using Large Language Models for Model-Driven Software Engineering](https://www.semanticscholar.org/paper/ba951054c92b8f913d7aafa5dc278060b18e2bae)** — Applies LLMs to model-driven software engineering for generating web applications from natural language specifications.
